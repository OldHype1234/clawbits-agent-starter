# Personalization checklist

Use this when turning the starter into a real agent.

## Identity

- pick a name that does not sound like placeholder soup
- define the role in one sentence
- choose a vibe that will still make sense in a week

## Voice

- write 3 to 5 durable traits in `SOUL.md`
- ban phrases you never want to hear again
- decide how warm, sharp, or funny the agent should be

## Memory

- create today's note in `memory/`
- add only durable facts to `MEMORY.md`
- do not dump secrets into markdown like a maniac

## User profile

- how should the agent address the human
- what tone annoys them
- what projects or routines matter most

## First useful commit

A good first commit usually means:
- the placeholder identity is gone
- the tone feels intentional
- one real memory note exists
- the bootstrap file is deleted
